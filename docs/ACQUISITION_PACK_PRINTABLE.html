3. Business Overview & Market

The Problem

Healthcare waiting rooms are chaotic. 87% of patients cite "wait times" as their #1 frustration. Clinics lose revenue when patients leave due to uncertainty.

The Solution: FilaZero

A fully digital orchestration layer.

  1. Patient: Scans QR code. Waits in car/coffee shop. Gets "Your Turn" alert on phone.
  2. Clinic: Manages flow via Reception Dashboard. Calls patients with 1 click.
  3. Waiting Room: TV Panel displays queue status automatically.

Market Size (TAM/SAM)

  • Global QMS Market: $1.2 Billion (growing 9.5% YoY).
  • Brazil Market: 200,000 private clinics.
  • Target: Small to medium clinics (3-10 doctors) who can't afford enterprise software ($1000/mo) but need order.

4. Technical Architecture

Technology Stack

  • Frontend: React 18.3, Vite 5, Tailwind CSS, Context API.
  • Backend: PocketBase 0.21 (Go + SQLite in WAL mode). Single binary.
  • Real-time: Native Server-Sent Events (SSE) via PocketBase. Sub-200ms latency.
  • Security: RBAC (Role Based Access Control), Sanitized Inputs, HTTPS.

Data Flow Diagram

[Client Device] <--> [Load Balancer/Nginx] <--> [PocketBase API] <--> [SQLite (WAL)]

*Real-time events pushed to all connected clients instantly.

Unique Offline Architecture

FilaZero uses a "Optimistic UI" approach with localStorage. If the clinic's internet drops, the Reception interface continues to work locally for critical actions, syncing when the connection is restored.

6. API Reference Overview

The system exposes a complete REST API via PocketBase.

Key Endpoints

  • GET /api/collections/tickets/records - List queue
  • POST /api/collections/tickets/records - Create new ticket
  • PATCH /api/collections/tickets/records/:id - Update status (Call/Finish)
  • GET /api/realtime - WebSocket stream for live updates

Note for Developer: Full Swagger/OpenAPI documentation is available in the /docs/technical folder of the repository.
